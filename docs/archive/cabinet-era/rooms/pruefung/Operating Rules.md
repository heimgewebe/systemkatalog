# Operating Rules

1. Jede Prüfung nennt Quellen, Commits und Zeitpunkt.
2. Deterministische Regeln laufen zuerst.
3. Ein Befund ohne Evidence bleibt ein Hinweis.
4. Wiederkehrende Befunde nutzen stabile Fingerprints.
5. Agentenausgaben sind Vorschläge.
6. Externe Modelle erhalten nur freigegebene, redigierte Inputs.
7. Jobs und Heartbeats bleiben vorerst deaktiviert.

## Prüfung-spezifisch
